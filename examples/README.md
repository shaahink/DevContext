# DevContext - Real Usage Examples

These are actual outputs generated by the current version of DevContext on real .NET projects, using the recommended `--task` + `--around` pattern.

## 01 - Add Reputation Feature (DntSite)

**Command:**
```bash
devcontext "I want to add a new feature for user reputation points and badges" \
  --around "src/DntSite.Web/Features/UserProfiles"
```

**Project:** Large feature-sliced production-ish Blazor app.

**Why this example is good:**
- Shows how the tool focuses when you have a clear feature intent.
- Produces relevant entities, services, endpoints, and mappers in the target area.
- Good layer summary for the UserProfiles bounded context.

## 02 - Understand a Method (CleanArchitecture)

**Command:**
```bash
devcontext "Explain how the CreateContributor endpoint + handler works, including domain entities, validation, and persistence it touches" \
  --around "src/Clean.Architecture.Web/Api/Contributors/Create.cs"
```

**Project:** Clean Architecture reference implementation.

**Why this example is good:**
- Demonstrates targeted extraction around a specific entry point (a single endpoint file).
- Pulls the relevant use case, domain, and infrastructure pieces without dumping the entire solution.

## How to Generate Similar Outputs

```bash
# Understand / debug a specific area
devcontext "your task or question here" --around path/to/relevant/folder/or/file.cs

# High-level architecture view
devcontext . --task "architecture review" --around src/YourBoundedContext
```

The more specific and intent-driven your `--task` + `--around` combination, the cheaper and more useful the resulting context becomes for LLMs.
